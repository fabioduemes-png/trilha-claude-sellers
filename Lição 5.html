O que são Projetos?

Explore cada conceito clicando nos itens abaixo. Projetos são muito mais do que pastas — são ambientes inteligentes.

Projetos são espaços de trabalho independentes com memória própria, histórico de conversas, base de conhecimento e instruções personalizadas. Pense neles como ambientes dedicados para fluxos de trabalho específicos — como ter uma sala de reunião virtual para cada área do seu trabalho.
Envie documentos relevantes uma única vez — o Claude os consulta em todas as conversas dentro do projeto. Sem precisar reenviar os mesmos arquivos toda vez. Suporta PDF, DOCX, CSV, TXT, HTML e muito mais.
Defina como o Claude deve se comportar: tom, nível de especialização, estilo de resposta, processos obrigatórios e requisitos específicos. Essas instruções se aplicam automaticamente a todas as conversas do projeto.
Quando a base de conhecimento se aproxima dos limites de contexto, o Claude ativa de forma transparente o modo de Geração Aumentada por Recuperação (RAG). Isso expande a capacidade em até 10x, recuperando de forma inteligente apenas as informações mais relevantes para cada pergunta.
Para usuários do Claude for Work, os projetos permitem colaboração com três níveis de permissão: Pode visualizar (acesso somente leitura), Pode editar (colaboração total) e Proprietário (controle completo). Compartilhe com indivíduos ou toda a organização.

Boas práticas

Dicas para usar Projetos

Navegue pelas dicas para aproveitar ao máximo os seus projetos.

01

Comece focado, depois expanda

Inicie com um caso de uso específico em vez de tentar criar um projeto para tudo. Você pode adicionar mais conteúdo ao longo do tempo conforme o projeto evolui.

02

Mantenha a base de conhecimento atualizada

Documentos desatualizados levam a respostas desatualizadas. Revise e atualize o conhecimento periodicamente para manter a qualidade.

03

Escreva instruções claras e específicas

Instruções vagas levam a resultados inconsistentes. Inclua contexto, tom, processo e requisitos explícitos nas instruções do projeto.

04

Nomeie arquivos de forma descritiva

O Claude usa os nomes dos arquivos para recuperar a informação correta. "Diretrizes-Marca-Q4-2024.pdf" é muito mais útil do que "documento1.pdf".

05

Faça referência aos documentos pelo nome

Ao fazer perguntas, mencione documentos específicos: "Com base no nosso relatório do Q3, quais foram as principais preocupações dos clientes?"

5 Qual a principal diferença entre Projetos e Skills?
Projetos são para equipes; Skills são para indivíduos
Projetos armazenam conhecimento; Skills executam tarefas
Projetos são gratuitos; Skills exigem plano pago
Projetos funcionam offline; Skills exigem internet
Projetos fornecem o "quê" (informação e contexto), enquanto Skills fornecem o "como" (processo e metodologia). Eles se complementam.
